dnl Command: CmdDelone
dnl

Command: compute Selling-reduced primitive cells
Purpose: Perform Selling/Delone reduction on input cells
Output type: S6 form of the reduced cells of input. The Delone-reduced
cell has only non-acute interaxial angles.
Parameters: NA

dnl -----------------------------------------------------------------------------------------------------------------------------------
dnl Command: CmdDists
dnl

Command: compute NCDist and CS6Dist distances
Purpose: For a group of input cells, distances are reported in 3 different calculations:
the simple Euclidean distance, the NCDist (G6) distance, and the CS6Dist
(S6) distances. Distances are calculated in 2 different ways. The first
is the rolling calculation of pairs in the list. The second is the distance of each
cell from the first input cell.
Output type: The distances for each input are output on a single line, starting with a
semicolon so that it will be ignored by succeeding programs. The input cells are echoed
to the output line as primitive and a, b, c &alpha;, &beta;, &gamma;, so that they
can be used for further input. Each output cell is followed
by its ordinal number, and the distance calculations list the ordinals for the
cells used of each calculation.
Parameters: NA

 end

dnl Command: CmdGen
dnl

Command: Generate cells of a particular type or types
Purpose: To Create valid unit cells of various user-chosen types
Output type: Output cells are reported in G6 form, randomly chosen cell
parameters for the various selected types. Lattice types are shown following
the numerical parameters.
Parameters: (default values in parentheses)
--- The number of cells of each type to be generated (1)
--- The type(s) of selected output requested. Examples are
--- "all" (default) -- produces examples of each of the 44 Niggli types
(see the International Tables) and also each of the 24 Delone types (all)
--- a number between 1 and 44 will generate examples of that Niggli type. For instance,
24 will create Niggli type 24 (which is hR). {allowed values: 1-44}
--- "m" generates monoclinic examples {allowed types: c, t, h, o, m, a}
--- "mC" generates C-centered monoclinic examples {mP, mS, mC, hR, hP, tP, tI, cF, cI, cP, aP}
--- "A1" generates a general triclinic Delone type with no S6 zero scalars.
--- "O4" generates the centered orthorhombic Delone type with signature {00r sst}.
--- "O" will generate all orthorhombic Delone types {allowed types: C, T, H, O, M, A}
--- Unrecognized input will generate all types

dnl Command: CmdLM
dnl

Command: apply Lattice Matching algorithm to listed cells
Purpose: Taking the first input cells as the "REFERENCE", the alternative cells of
each succeeding cell are examined to find a closest match to the reference lattice.

Andrews, Lawrence C., Herbert J. Bernstein, and Nicholas K. Sauter.
"Approximating lattice similarity." Acta Crystallographica Section A: Foundations and Advances 79.5, 480-484 (2023).

Output type: a, b, c &alpha;, &beta;, &gamma;, with the lattice centering
chosen to be the same as that of the reference cell.
Example:
f 10 10 10 90 90 90
p 7 7.1 6.9 59 60 61
end
; Lattice Matching
F 10.000 10.000 10.000 90.000 90.000 90.000 ( 0 degrees in S6) REFERENCE
F 10.050 9.821 9.832 89.143 92.094 88.781 ( 3.07179 degrees in S6)
end
Parameters: NA

dnl Command: CmdNiggli
dnl

Command: compute Niggli-reduced primitive cells
Purpose: Perform Niggli reduction on input cells. The result has
the shortest edge lengths of the lattice and is the "most orthogonal" unit cell.
Output type: G6 form of the reduced cells of input
Parameters: NA

dnl -----------------------------------------------------------------------------------------------------------------------------------
dnl Command: CmdPath
dnl

Command: compute path between pairs of cells
Purpose: A series of points is computed between each pair of input cells. The
computation is done in S6. For n input cells, n-1 paths will be
generated. The individual points are labeled with their ordinals and each
segment is labeled (after the 6 parameters).
Output type: S6
Parameters: The number of points in -each- path (20)

dnl -----------------------------------------------------------------------------------------------------------------------------------
dnl Command: CmdSella
dnl

Command: apply Sella algorithm
Purpose: Determine potentially higher symmetry for input cells
Output type: agreement factors for each of the Bravais lattice types
that fill well and reduced cell information for the input cells. Also the name
of the graphical output file.
Graphical output: the figure of the hierarchy of Bravais lattice types
with numerical agreement determined by Sella.

Grimmer, H., & Nespolo, M. (2006).
Geminography: the crystallography of twins. Zeitschrift
für Kristallographie-Crystalline Materials, 221(1), 28-50.)

Parameters: choices if only some outputs are needed (all)

dnl Command: CmdToB4
dnl

Command: compute Bravais tetrahedron (B4)
Purpose: computes the edge vectors of the unit cell and their negative sum. These 4
vectors also define the Bravais tetrahedron Delone, B. N., Galiulin, R. V.,
& Shtogrin, M. I. (1975). On the Bravais types of
lattices. Journal of Soviet Mathematics, 4(1), 79-156.
Output type: echoes the input cell as a, b, c, &alpha;, &beta;, &gamma;,
followed by the vector representations of the unit cell edges and their vector sum,
and then a listing of their lengths.
Parameters: NA

dnl Command: CmdToS6L
dnl

Command: compute linearized S6 or Root Invariant version of cells
Purpose: Both of the output choices are 6-space vectors of positive numbers.
The values are derived by first applying Selling/Delone reduction so that all of
the S6 values are non-positive. Then the square roots of the negative
of those values are used to create the result (linearized S6). The
root invariant is generated by applying sorting rules of Bright, Cooper, and Kurlin.

Bright, M., Cooper, A. I., & Kurlin, V. (2021).
Welcome to a continuous world of 3-dimensional lattices.
arXiv preprint arxiv:2109.11538, 2.

Output type: The type followed by the parameters.
Parameters: "all" or blank for output of both types, "R" for root invariant, and S for linearized S6

dnl

Command: compute unsorted Dirichlet cells (dc7unsrt)
Purpose: convert to the 7 parameter form related to the Dirichlet
(also known as Vornoi or Wigner-Seitz) unit cell. (note that lattice centering is not removed)
Bernstein and Andrews, 2023.
Output type: DC7u

dnl -----------------------------------------------------------------------------------------------------------------------------------
dnl Command: CmdToV7
dnl

Command: compute V7 version of cells
Purpose: convert to the 7 parameter vector of Andrews, Bernstein, Pelletier, 1980.
The vector is define in terms of the Niggli reduced cell a,b,c and the Niggli reduced
reciprocal cell 1/a*, 1/b*, 1/c*, and the cube root of the reduced cell volume.
(note that lattice centering is not removed). Note that all 7 scalars are in the units of Angstroms.
Output type: V7
Parameters: NA

dnl -----------------------------------------------------------------------------------------------------------------------------------
dnl Command: PlotC3
dnl

Command: draw C3 plot of listed cells
Purpose: Graphical output of multiple unit cells in C3
Output type: Name of the file containing the plots
Graphical output: 2-D plots of the 3 complex coordinates in C3
Parameters: NA
The 3 (2-D) coordinates are plotted in a single row. The all-negative quadrants
are shown. A consequence is that if the unit cells have not been Selling/Delone
reduced, then points will plot outside of that quadrant.
