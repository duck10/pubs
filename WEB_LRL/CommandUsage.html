dnl Command: CmdDists
dnl

Command: compute NCDist and CS6Dist distances
Purpose: For a group of input cells, distances are reported in 3 different calculations:
the simple Euclidean distance, the NCDist (G6) distance, and the CS6Dist
(S6) distances. Distances are calculated in 2 different ways. The first
is the rolling calculation of pairs in the list. The second is the distance of each
cell from the first input cell.
Output type: The distances for each input are output on a single line, starting with a
semicolon so that it will be ignored by succeeding programs. The input cells are echoed
to the output line as primitive and a, b, c &alpha;, &beta;, &gamma;, so that they
can be used for further input. Each output cell is followed
by its ordinal number, and the distance calculations list the ordinals for the
cells used of each calculation.
Parameters: NA

dnl

Command: Generate cells of a particular type or types
Purpose: To Create valid unit cells of various user-chosen types
Output type: Output cells are reported in G6 form, randomly chosen cell
parameters for the various selected types. Lattice types are shown following
the numerical parameters.
Any input lattice descriptions are copied to the output. Parameters: (default values in parentheses) (deprecated)
--- The number of cells of each type to be generated (1)
--- The type(s  of selected output requested. See Control Parameters

Control Parameters
Control information is case sensitive.
COUNT: the number of each chosen type to randomly generate (default: 5)
TYPE:
Examples are:
  • --- "all" (default  -- produces examples of each of the 44 Niggli types (see the International Tables  and also each of the 24 Delone types (all)
  • --- "Gruber" (default  -- produces the the requested number of copies of the five nearly-Niggli-reduced
    examples of Gruber, 1973.
  • --- "random" (default  -- produces the the requested number of random unit cells.
  • --- a number between 1 and 44 will generate examples of that Niggli type. For instance,
    24 will create Niggli type 24 (which is hR). {allowed values: 1-44}
  • --- "m" generates monoclinic examples {allowed types: c, t, h, o, m, a}
  • --- "mC" generates C-centered monoclinic examples {mP, mS, mC, hR, hP, tP, tI, cF, cI, cP, aP}
  • --- "A1" generates a general triclinic Delone type with no S6 zero scalars.
  • --- "O4" generates the centered orthorhombic Delone type with signature {00r sst}.
  • --- "O" will generate all orthorhombic Delone types {allowed types: C, T, H, O, M, A}
  • --- a list of types will generate each of the types; example: "44 O3 H4 C5"
  • --- Unrecognized input will generate all types

dnl Command: CmdLMP3
dnl

Command: CmdLMP3 Lattice matching in 3D using P3 distances for measuring fit.
Purpose: Lattice matching that finds transformation matrices between crystal lattices
The first cell input is treated as the reference that following "mobiles" will be
matched to. The output includes the P3 distances and S6 angle measures of fit.
If more than one mobile is input, then "comparison mode" is output, which
is more compact. Output include the transformation matrices that operate on
the base vectors of the unit cell (this is important). Note, there are likely
to be more than one transformation listed, but they are listed in order of
of worsening fit.
Output type: transformation matrices, distance metrics (P3 and S6 angles), and transformed
lattice parameters with detailed quality assessment and multiple solution rankings.
Example:
p 10.25 10.74 21.08 87.72 75.97 61.53
p 10.25 10.74 21.08 78.96 75.97 61.49
end

  S6 angle: 0.01°
  Transformed: 10.250 10.739 21.080 87.714 75.970 61.502

Parameters: NA
Control Parameters
Control information is NOT case sensitive, and control names can be abbreviated.
COMPARISONMODE: enable comparison mode for multiple input processing (default: true)
TEST: run test suite to verify system functionality
 TEST (no number): run all available tests
 TEST n: run specific test number n
 TEST -1 (or any negative): show list of available tests
 TEST 999: show list of available tests

Test Suite
The system includes a comprehensive test suite with the following tests:
 Test 1: Hawaiian- Nearly identical cells with known transformation

 Test 7: DuplicateDetection - Ensure no duplicate transformation matrices in results
 Test 8: HighSymmetryDuplicates - High symmetry case prone to generating duplicates
 Test 9: MultipleValidTransforms - Case where multiple valid transformations exist

Usage:

dnl

Command: apply Lattice Matching algorithm to listed cells
Purpose: Taking the first input cell as the "REFERENCE", the alternative cells of
each succeeding cell are examined to find a closest match to the reference lattice.
Matching is done in the space S6, so 3D transformations are not computed.

Andrews, Lawrence C., Herbert J. Bernstein, and Nicholas K. Sauter.
"Approximating lattice similarity." Acta Crystallographica Section A: Foundations and Advances 79.5, 480-484 (2023).

Output type: a, b, c &alpha;, &beta;, &gamma;, with the lattice centering
chosen to be the same as that of the reference cell.
Example:
f 10 10 10 90 90 90
p 7 7.1 6.9 59 60 61
end

end

dnl Command: CmdToRI
dnl

Command: compute RI (Root Invariant) version of cells
Purpose: convert to RI representation (note that lattice centering is not removed)
Output type: RI
Parameters: NA

Root invariant (RI) is a lattice representation introduced by Bright and Kurlin (BK) in several papers. RI is best understood as a reordering of the Selling-reduced scalars (space S6, s1,s2, s3, s4, s5, s6).

Step 1: Create a new vector p with six elements that the negatives of the Selling-reduced S6 vector;

Step 2: Revise p by taking the square root of each element.

Step 3: Exchange the columns of p to place the column that has the small scalar into the first column.

Step 4: If the smallest element is in the second row, exchange the first row and second row of that column, and do the same for one of the other columns.

Step 5: Using similar logic, place the smallest scalar of the new columns 2 and 3 into the second column and in the first row.

Step 6: If the first row scalars of columns 1 and 2 are equal, then exchange those columns such that the smaller second row scalar is in column 1. There is no need to specify the result for the case where the second row scalars are equal, because the results would be identical.


Above is the logic for the case described as V1 by BK, where V1..V5 refer to the “Voronoi polyhedron” types. V1 is the most general case, with no 90 degree angles in the reduced cell. The V1 polyhedra have 14 sides. BK also describe separate root invariants for each of the Voronoi types, where each RI has fewer than 6 scalars.

dnl Command: CmdToS6L
dnl

Command: compute linearized S 6 or Root Invariant version of cells
Purpose: Both of the output choices are 6-space vectors of positive numbers.
The values are derived by first applying Selling/Delone reduction so that all of
the S6 values are non-positive. Then the square roots of the negative
of those values are used to create the result (linearized S6). The
root invariant is generated by applying sorting rules of Bright, Cooper, and Kurlin.

Bright, M., Cooper, A. I., & Kurlin, V. (2021).
Welcome to a continuous world of 3-dimensional lattices.
arXiv preprint arxiv:2109.11538, 2.

Output type: The type followed by the parameters.
Parameters: "all" or blank for output of both types, "R" for root invariant, and S for linearized S6

dnl

Command: Follow
Purpose:
In the default mode (POINT), a single cell is read, and its Niggli-reduced cell is calculated.
Multiple cells can be read, and a graphical output is computed for each input cell.
In space S6, a straight line is computed between those points. Then the distances
from points (default: 100 points  along that line are calculated to the end point (Niggli-reduced cell,
in the POINT case). The graphical output shows those distances for each chosen
distance measure (default: NCDist in G6 and CS6Dist in S6).
For more details on the Follower method, see Andrews and Bernstein:
"The Follower algorithm and a program using it to explore spaces"
Output type: graphical file names and various text
Graphical output:Follow consumes unit cells descriptions.
